# 📊 图计算可视化系统 (Graph Computing System)

一个基于 Web 的轻量级高性能图数据可视化与分析平台。专为复杂网络分析设计，支持交互式图谱编辑、核心指标计算、社区发现算法以及多格式数据（CSV/JSON）的导入导出。

## 🛠️ 第一部分：开发者部署指南

如果你是开发者，或者拿到了项目源码想要在自己的电脑上运行，请严格按照以下步骤从零开始配置。

### 1. 环境准备 (Prerequisites)

本项目基于现代前端技术栈 (React + Vite)，运行需要依赖 **Node.js** 环境。

#### 第一步：下载并安装 Node.js (必选)
1.  访问官网：[https://nodejs.org/zh-cn/](https://nodejs.org/zh-cn/)
2.  下载并安装 **LTS (长期支持版)**，这是一路“下一步”的傻瓜式安装。
3.  **验证安装**：
    *   按下 `Win + R`，输入 `cmd` 并回车。
    *   在黑窗口输入 `node -v`。
    *   如果显示版本号（如 `v18.16.0`），说明安装成功。

#### 第二步：准备编辑器
推荐使用 **Visual Studio Code (VS Code)** 来管理和运行代码。
*   下载地址：[https://code.visualstudio.com/](https://code.visualstudio.com/)

### 2. 获取与安装项目

#### 第一步：下载代码
你可以选择以下任意一种方式：
*   **方式 A (推荐)**：使用 Git 克隆
    ```bash
    git clone https://github.com/One0f/graph-computing-system.git
    ```
*   **方式 B**：下载压缩包
    点击 GitHub 页面绿色的 **Code** 按钮 -> **Download ZIP**，下载后解压到电脑任意文件夹。

#### 第二步：打开项目
1.  启动 VS Code。
2.  点击左上角 **文件 (File)** -> **打开文件夹 (Open Folder)**。
3.  选择你刚才解压的项目根目录 `graph-computing-system`。

#### 第三步：安装依赖 (最重要的一步)
刚下载的代码是不包含第三方库的，你需要执行命令让它自动下载。

1.  在 VS Code 中，点击顶部菜单栏 **“终端 (Terminal)”** -> **“新建终端 (New Terminal)”**。
2.  在下方的终端窗口中，输入以下命令并回车：

    ```bash
    npm install
    ```

3.  等待进度条走完。如果显示 `added xxx packages`，说明安装成功。

## 📖 第二部分：用户功能手册

### 🖥️ 1. 界面概览
*   **左侧控制台**：所有功能的入口，包括搜索、编辑、文件操作和算法分析。
*   **右侧画布**：图谱展示区。支持鼠标滚轮缩放画布、按住左键拖拽画布、鼠标点击拖拽节点。

### 🔍 2. 搜索功能
*   在左上角输入框输入 **节点ID** 或 **节点名称**。
*   点击 🔍 图标或按回车。
*   **效果**：视图自动飞跃并聚焦至目标节点，目标节点呈现 **红色光晕高亮**，其余无关节点透明度降低。
*   点击 ❌ 图标：清除搜索结果，重置画布到全局视角。

### ✏️ 3. 图谱编辑 (Editor)
*   **+ 节点**：在画布中心区域创建一个新的空白节点。
*   **🔗 连线**：
    1.  点击按钮进入“连线模式”（按钮变黄）。
    2.  依次点击画布上的【起点】和【终点】。
    3.  系统自动生成一条带箭头的有向边。
*   **🗑️ 删除**：选中任意节点或连线（选中项显示深色边框）后，点击删除按钮。

### 💾 4. 数据导入导出

#### JSON 格式 (全量存档)
*   **JSON 导出**：将当前的画布状态（节点位置、颜色、大小、关系）完整保存为一个 `.json` 文件。
*   **JSON 导入**：读取存档文件，100% 还原上次的工作现场。

#### CSV 格式 (Excel 对接)
*   **📥 导 CSV 节点**：导入节点列表。
    *   **逻辑**：系统会自动询问是“清空画布重新导入”还是“保留旧数据追加新数据”。
*   **📥 导 CSV 关系**：导入关系列表。
    *   **智能补全**：如果 CSV 关系表中引用了不存在的节点 ID，系统会**自动创建缺失的节点**（显示为黄色虚线框），确保连线能画出来。

> **CSV 格式要求 (建议使用 UTF-8 编码)**：
> *   **节点表**：系统会自动识别 `id` (编号) 和 `label` (名称) 列。
> *   **关系表**：系统会自动识别 `source` (起点) 和 `target` (终点) 列。支持中文列名如“项目编号”、“参建单位”。

### 🔬 5. 核心图算法 (Analytics)

系统集成了基于 `Graphology` 的 5 大核心算法：

| 图标 | 算法名称 | 功能与用途 |
| :--- | :--- | :--- |
| 🕸️ | **整理布局** (Layout) | **ForceAtlas2 引擎**：模拟物理引力和斥力。让重叠、杂乱的节点自动弹开，形成舒展、无交叉的拓扑结构。 |
| 🔴 | **度中心性** (Degree) | **寻找“大V”**：根据连接数量计算。连接越多的节点，颜色越深红，体积越大。用于识别网络中的热门枢纽。 |
| 🌉 | **介数中心性** (Betweenness) | **寻找“桥梁”**：计算节点在中介传递中的作用。核心桥梁节点会显示显著的**紫色双重边框**。用于识别连接两个圈子的关键人物。 |
| 🎨 | **社区发现** (Louvain) | **寻找“团伙”**：自动计算网络中的社团结构，并将同一社团的节点染成相同颜色。用于分析组织架构或团伙划分。 |
| ⭐ | **PageRank** | **寻找“权威”**：Google 排名算法。不仅仅看连接数，更看被谁连接。评估节点在整个网络中的权威度。 |
| 🛤️ | **最短路径** (Shortest Path) | **路径导航**：点击开启模式 → 依次点选 A点 和 B点。系统会用**红色粗线**高亮显示两点间的最短链路。 |

### 🛠️ 6. 辅助功能
*   **重置样式**：清除所有算法产生的颜色和高亮，将节点恢复到默认的简洁黑白样式，但不删除数据。
*   **恢复数据 (Hello World)**：**慎用！** 一键清空当前画布，并加载系统内置的 6 节点演示数据。
